# OpenCode Adapter

This adapter binds the runner-neutral capability pack to OpenCode through official extension surfaces.

It must remain disposable: if OpenCode changes internals, this folder should absorb the change while `contracts/` and `capabilities/` remain stable.

## Preferred Official Surfaces

- plugins for event observation and guardrails
- custom tools for state bridge calls
- skills for behavior instructions
- agents/subagents for bounded roles
- permissions for edit/shell/task containment
- compaction hook for restore packet injection
- `shell.env` hook for safe session-local env additions

## Adapter Rules

1. Use project/session-local OpenCode config generated by MMF/MMS; do not write global OpenCode config by default.
2. Treat OpenCode event names and hook names as adapter-version facts, not neutral contract names.
3. Custom tools may call bridge APIs, but must not directly mutate canonical task-state storage.
4. Agent definitions should be generated from role slots and profile bindings, not from hardcoded historical agent names.
5. Permissions must default conservative: read-only for explore/review/vision, ask for executor/fixer edits, deny destructive operations unless the profile explicitly allows them.
6. Adapter health checks should verify presence/version/manifest compatibility only; they must not decide workflow truth.

## MVP Binding Map

| Neutral contract | OpenCode binding candidate | Notes |
| --- | --- | --- |
| `session_start` | plugin event observation + generated session manifest | Do not assume internal storage shape. |
| `before_user_turn` | plugin event/tool guard path where available | Fall back to explicit tool invocation if no stable hook exists. |
| `subagent_dispatch` | generated agents + permissions + task/subagent call discipline | Payload should be `contract_ref` or budgeted projection. |
| `finish_request` | custom tool requesting state-core done/block transition | Tool asks; state-core decides. |
| `session_restore` | compaction hook restore packet | Keep packet minimal and non-secret. |
| `shell.env` | safe session-local env injection | No real-home auth or credential copying. |

## First Spike

Before full implementation, prove these six OpenCode surfaces in a local spike:

1. plugin event observation
2. `tool.execute.before` and `tool.execute.after` guard hooks
3. `shell.env` session env injection
4. compaction restore injection
5. custom tool execution
6. generated agent permission boundaries
